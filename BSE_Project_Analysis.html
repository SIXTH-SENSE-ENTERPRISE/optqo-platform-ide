BSE Project Analysis

A comprehensive analysis of the Bombay Stock Exchange (BSE) data processing project

1. Project Overview

The BSE Project is a sophisticated data processing and analysis system designed to handle financial data from the Bombay Stock Exchange. The project focuses on data quality management, entity deduplication, announcement analysis, and financial pattern detection. It uses a combination of SAS (Statistical Analysis System) and SQL to process and analyze large volumes of financial data efficiently.

2. Project Structure

The project consists of multiple components organized in a modular fashion:

2.1. SAS Scripts

  • 1st Part.sas - Initial data processing script that processes announcements from the Bombay Stock Exchange and categorizes them based on keywords.
  • SAS Code for Revised Insider Daily PAN.txt - Script for processing Permanent Account Number (PAN) data for insider trading analysis.

2.2. SQL Jobs

  • JOB REJECT INVALID DATA INCR_2.sql - Data validation job that filters out records with invalid or incomplete data.
  • JOB ASSIGN CLUSTER ID INCR_4.sql - Assignment of cluster IDs to group similar entities based on match codes.
  • LOOP JOB STAND MATCHCODE INCR _3.sql - Parallel processing job that standardizes data fields and generates match codes for deduplication.

2.3. Query Files

  • 2ndQuery.txt, 3rdQuery.txt, 4thQuery.txt, 5thQuery.txt - Various query scripts for data analysis and processing.
  • 4thQuery_part1.txt through 4thQuery_part6.sql - Components of a large query broken into manageable parts.

2.4. Documentation

  • SAS_Code_Understanding.xls - Excel documentation explaining SAS code functionality.
  • README.md - Basic repository information.

3. Core Functionality Analysis

3.1. Data Cleaning and Standardization

The project employs sophisticated data cleansing techniques:

  • Name Standardization - Names are standardized using DQSTANDARDIZE functions with specialized rules for both individuals and organizations.
  • Address Standardization - Addresses are cleaned and formatted consistently to enable reliable matching.
  • PAN Validation - Permanent Account Numbers are validated against expected patterns and standardized for consistent usage.
  • Geographic Standardization - City, state, and country names are standardized using reference datasets.

3.2. Deduplication Process

The project uses a multi-layered approach to identify duplicate entities:

  1. Match Code Generation - Creates standardized tokens for comparison using SAS Data Quality functions like DQMATCH.
  2. Multiple Match Strategies - Employs up to 6 different matching algorithms (DEDUPE_MATCH_CODE_1 through DEDUPE_MATCH_CODE_6) using different field combinations:
    • Name + PAN
    • Address + PAN
    • Other combinations based on data availability
  3. Clustering - Related records are grouped together using cluster IDs.
  4. Match Flagging - Each match code has an associated flag (DMC_1_FLAG through DMC_6_FLAG) indicating validity.

3.3. Announcement Data Processing

Financial announcements are processed using natural language techniques:

  • Keyword Extraction - Identifying terms like "dividend," "merger," "acquisition," "bonus," etc.
  • Sentiment Classification - Categorizing announcements as favorable, unfavorable, or neutral.
  • Impact Assessment - Analyzing price and volume variations following announcements.

3.4. Parallel Processing

The project implements efficient parallel processing to handle large datasets:

  • Data Partitioning - Records are divided into chunks for parallel processing.
  • Grid Computing Support - Utilizes SAS Grid computing when available.
  • Load Balancing - Distributes work across available CPU cores.
  • Process Monitoring - Tracks execution across parallel tasks.
